 [1-1] S3 란?
  Simple Storage Service
  >>> 객체 기반의 무제한 파일 저장 스토리지
  >>> URL을 통해 손쉽게 파일 공유 가능
  >>> 정적 웹사이트 호스팅 가능

 [1-2] S3 사용 용도
  >>> 주로 수정이 잘 일어나지 않는 최종 파일을 저장
  >>> 빠른 속도로 Read/Write을 해야 하는 경우 부적합
  >>> 적합한 용도
      : 미디어 파일 보관(이미지,동영상,음악,문서)
      : 로그 파일 보관(Logtash,Hadoop 등과 연동)
      : 데이터 백업(다른 AWS 서비스의 데이터 백업)
      : 정적 웹 호스팅(HTML,JS,CSS)

 [1-3] S3 관련 용어
  >>> Bucket : Object를 담는 최상위 단위 (이름,개수 제한 있음)
  >>> Object : Bucket 에 담는 데이터의 단위(파일)
  >>> Metadata : Obkect 에 대한 여러가지 정보를 담고있는 데이터
  >>> Policy : Bucket, Object 에 대한 접근을 통제하는 권한 정보

  [1-4] S3 사용 방법
  버킷 생성 -> HTTP로 버킷에 파일 업로드(오브젝트로 저장됨) -> 오브젝트에 권한 설정 -> HTTP로 버킷 파일 다운로드

  [1-5] S3 스토리지 클래스
  >>> 자주 액세스하는 객체를 위한 스토리지 클래스
  - standard : 기본 스토리지 클래스
             : 높은 내구성, 가용설 및 성능을 갖춘 객체 스토리지를 제공

  >>> 자주 액세스하지 않는 객체를 위한 스토리지 클래스
  - standard-IA (Infrequent Access)
    : 여러 개의 가용영역에 중복 저장(고가용성, 복원력 있음)
  - One Zone-IA (Infrequent Access)
    : 하나의 가용 영역에만 저장(Standard-IA 비해 저렴, 복원력 없음)

  [1-6] S3 Glacier
  >>> 아카이빙 백업 스토리지
  >>> 무제한 확장성
  >>> 백업 및 아카이빙 용도의 Cold 데이터

  [1-7] S3 객체 라이프 사이클 관리
  >>> 스탠다드에 처음 들어온 파일 ----(30일 경과)---> S3 Standard-IA ----(30일 경과)---> S3 Glacier Flexible Retrieval
                                   45 % 저렴                                67 % 저렴

  >>> 글래시어는 보관용으로 생각

- 파일 스토리지
  >>> 데이터를 파일 단위로 관리
  >>> 파일 내용 일부분만 수정 가능
  >>> 운영체제에서 직접 접근 가능(볼륨 마운트)

- 객체 스토리지
  >>> 데이터를 객체 단위로 관리
  >>> 객체 데이터 일부분만 수정 불가능
  >>> 운영체제에서 직접 접근 불가
- 블록 스토리지
  >>> 데이터를 블록 단위로 저장
  >>> 엄청나게 낮은 지연 시간을 제공
  >>> 적합한 사용 사례
